QA Section

javascript is a client-side scripted language. javascript code runs in the browser with the help of the browser engine. The code is parsed and eventually converted to machine code by the browser engine to run on the CPU. Again, this is a single-threaded language that can do a single task at a time. All the tasks are listed in the call stack and the synchronous tasks start executing while the asynchronous tasks are first queued in webapis and then transfered to the callback queue. When the call stack is empty, the event loop triggers and transfers tasks from the callback queue one after another to the call stack to eventually execute. Thus javascript works.

We know, javascript can do one task at a time, as a result, the asynchronous tasks are first queued in the webapis and then transferred to the callback queue. The event loop handles this callback queue. It has a single task to do and that is, when the call stack is empty, it pushes tasks from the callback queue to the call stack in order to execute. It has to wait until the call stack is empty. After the event loop triggers, it also lets the render queue render which indicates that the event loop is directly related to a web page's fluidity & smooth responsiveness.
